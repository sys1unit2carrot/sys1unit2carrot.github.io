Jekyll導入について
Jekyllとは

Github ioでWEbサイトを作成する際にRubygemsとして配布されているJekyll(ジキル)を導入するとマークダウンのファイルをHTMLに変換してくれるので、記事を簡単に投稿できるらしい。

参考：http://jekyllrb-ja.github.io/

調査した導入手順

1. ruby導入

ruby 2.4.5p335 (2018-10-18 revision 65137) [x86_64-linux]　導入済み


2. rbenv導入

rbenv 1.1.2-34-g0843745　導入済み


3. bundler導入

Bundler version 2.2.30　導入済み


4. Jekyll導入

gem install jekyllを実行

jekyll -vで導入されたバージョンを確認

jekyll -v
jekyll 4.2.1

5. jekyllでプロジェクトを立ち上げる

jekyll new [Githubのリポジトリ名]を実行


6. ディレクトリ移動

cd [Githubのリポジトリ名]を実行


7. サーバー起動

bundle exec jekyll sを実行


8. ターミナルに表示されたURLにアクセスして表示確認

→アクセスできない！

調査した内容

・オプション追加してjekyll sを実行

VagrantでJekyllを動かしてもlocalhostで接続できないのサイトを参考に以下のコマンドを実行したが状況は変わらず。

bundle exec jekyll s --host 0.0.0.0

・ポート番号を変更

以下のコマンドを実行してポート番号を変更したが変わらず。

bundle exec jekyll s -P 5000

・URLの末尾にhtml名を記載

http://localhost:4000/index.htmlのように、URLの末尾にhomeとして表示するHTML名を記載してアクセスしたが変わらず。
